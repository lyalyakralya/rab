# ПР №6. Анализ сетевого трафика: tcpdump и Wireshark

## 1. Разбор строки tcpdump

Строка: `12:34:56.789012 IP 127.0.0.1.44321 > 127.0.0.1.8080: Flags [S], seq 123, length 0`

| Поле | Значение | Что означает |
|------|----------|---------------|
| 12:34:56.789012 | 12:34:56.789012 | Время захвата пакета |
| IP | Internet Protocol | Протокол сетевого уровня |
| 127.0.0.1.44321 | 127.0.0.1:44321 | IP-адрес и порт источника (локальный) |
| 127.0.0.1.8080 | 127.0.0.1:8080 | IP-адрес и порт назначения |
| Flags [S] | SYN-флаг | Начало TCP-соединения (SYN-пакет) |
| length 0 | 0 байт | Нет полезной нагрузки (только заголовки) |

## 2. Перехваченные данные HTTP

![Пароль в открытом виде](screens/4-http-password-plaintext.png)

**Что видно в трафике:** Логин и пароль передаются в открытом виде: `username=admin&password=supersecret123`

**Кто в реальной сети может прочитать трафик:** Злоумышленник в той же сети (Wi-Fi, коммутатор с портом мониторинга), администратор сети, провайдер.

## 4. HTTP vs HTTPS — сравнение

| Что наблюдаем | HTTP | HTTPS |
|---------------|------|-------|
| IP-адреса сторон | Видны | Видны |
| Порт назначения | Виден (8080) | Виден (443) |
| Заголовки HTTP | Видны открыто | Зашифрованы |
| Тело запроса | Видно открытым текстом | Зашифровано |
| Логин и пароль | Видны | Не видны |
| Имя домена (SNI) | Видно в Host | Видно в SNI (утекает) |

## 5. DNS и приватность

**Какие домены видны в DNS даже при HTTPS:** github.com, example.com, google.com

**Что это означает:** Злоумышленник видит, какие сайты вы посещаете, даже если трафик зашифрован. DNS-запросы не шифруются по умолчанию.

## 6. Base64

**Декодированная строка:** `dmFzeWE6TXlQYXNzd29yZDIwMjQ=` → `vasya:MyPassword2024`

**Почему Base64 не является шифрованием:** Это просто кодирование (трансформация) без использования ключа. Любой может декодировать Base64 обратно. Настоящее шифрование требует ключа.

## 7. tshark vs tcpdump vs Wireshark

| Инструмент | Когда использовать |
|------------|--------------------|
| tcpdump | Быстрый захват на сервере, минималистичный, работает везде |
| tshark | Автоматизация, анализ на серверах без GUI, фильтрация |
| Wireshark | Глубокий анализ, визуализация, удобный GUI |

## 8. Каналы утечки через сеть

**Выявленные каналы утечки:**
- HTTP (незашифрованный) — пароли и данные открыты
- DNS — видно какие сайты посещаются
- IP-адреса — видны всегда

**Меры защиты:** 
- Использовать HTTPS везде
- Использовать DNS-over-HTTPS (DoH) или DNS-over-TLS (DoT)
- VPN для скрытия IP-адресов
