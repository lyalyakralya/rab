## 6. Контрольные вопросы

**68. В чём разница между шифрованием и подписью? Можно ли одновременно зашифровать и подписать файл?**

**Ответ:** 
- Шифрование — защищает содержимое файла от просмотра посторонними. Тот, у кого есть приватный ключ, может расшифровать и прочитать данные.
- Подпись — подтверждает авторство и целостность файла. Подпись не скрывает содержимое.
- Да, можно одновременно зашифровать и подписать: `gpg --sign --encrypt --recipient EMAIL file`

**69. Почему публичный ключ можно раздавать всем, а приватный нельзя никому?**

**Ответ:**
- Публичный ключ предназначен для шифрования и проверки подписей. Даже если он попадёт к злоумышленнику, он не сможет расшифровать данные или создать подпись от вашего имени.
- Приватный ключ нужен для расшифровки и создания подписей. Если он попадёт к другому человеку, он сможет расшифровать ваши секретные данные и подписываться от вашего имени.

**70. Что произойдёт с зашифрованными данными если потерять приватный ключ?**

**Ответ:**
- Данные будет невозможно расшифровать. Они станут навсегда недоступными.
- Поэтому важно хранить резервную копию приватного ключа в безопасном месте.

**71. Можно ли подделать Verified коммит на GitHub не имея приватного GPG-ключа владельца?**

**Ответ:**
- Нет, нельзя. Подпись коммита создаётся с помощью приватного ключа.
- GitHub проверяет подпись с помощью публичного ключа, который загружен в настройках аккаунта.
- Без приватного ключа невозможно создать корректную подпись, которую GitHub признает Verified.

**72. Почему shred плохо работает на SSD? Что использовать вместо него?**

**Ответ:**
- На SSD используется технология wear leveling (выравнивание износа) и TRIM. Данные могут быть физически перезаписаны в других ячейках, а оригинальные блоки остаться с данными.
- Вместо shred на SSD рекомендуется:
  - Использовать шифрование всего диска (LUKS)
  - Использовать команду `blkdiscard` для TRIM
  - Использовать утилиты производителя SSD для безопасного удаления

**73. Коллега говорит: «я отформатировал диск перед продажей — всё безопасно». Он прав?**

**Ответ:**
- Нет, он не прав. Быстрое форматирование удаляет только метаданные (таблицу файлов), но данные остаются на диске.
- Их можно восстановить с помощью специальных утилит (testdisk, photorec).
- Для безопасной продажи нужно использовать `shred`, `dd if=/dev/urandom` или полное шифрование диска.

**74. Чем отличается detach-sign (--detach-sign) от встроенной подписи (--sign)?**

**Ответ:**
- `--detach-sign` — создаёт отдельный файл подписи (файл.asc). Исходный файл остаётся неизменным.
- `--sign` — встраивает подпись внутрь файла (создаёт файл.gpg). Исходные данные и подпись находятся в одном файле.
- Отдельная подпись удобна для распространения: файл и подпись передаются отдельно.
